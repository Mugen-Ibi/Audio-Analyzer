# Pro Audio Analyzer

音響エンジニアが空間やスピーカーシステムの特性を正確に測定・評価するための、フルRustアーキテクチャによるプロフェッショナル向け高機能オーディオアナライザです。

現在、**Phase 3（リアルタイム・スペクトラムアナライザ）** まで実装されています。

## 1. 哲学 (Philosophy)

このプロジェクトは、単なる「波形観察アプリ」ではありません。現場の暗騒音下においても、数学的に裏付けられた「位相」と「振幅」を低レイテンシかつ確実に可視化するための「真の測定器」です。

利便性や開発のしやすさを優先し、測定データに不確実性を混入させる既存の常識を私たちは拒絶します。OSによる暗黙のリサンプリング、予測不可能なガベージコレクション、描画ツリーのオーバーヘッドは、すべて排除されるべきデータ汚染の源泉です。

## 2. 設計の第一原理 (First Principles of Architecture)

本プロジェクトは以下の絶対的な制約のもとに構築されています。

- **OSミキサーの完全バイパス (ASIO Native):** Windows標準のオーディオスタックによる介入を物理的に回避します。可能な限りASIOドライバーを優先してハードウェアを直接叩き、WASAPIへフォールバックした場合もデバイスのネイティブ・サンプリングレートとバッファサイズを強制します。
    
- **ガベージコレクション（GC）の排除:** ミリ秒単位の確実性が求められるDSP処理において、実行時間を予測できないGCを伴う言語は採用しません。メモリ管理をコンパイラレベルで保証するRustのみを使用します。
    
- **スレッドロックの禁止:** DSPスレッド内でのミューテックスによるロックや、動的なヒープアロケーションを固く禁じます。オーディオI/OとUIスレッド間の通信は、完全にロックフリーなリングバッファ（Lock-free Ring Buffer）によって行われます。
    
- **Retained Mode GUIの放棄:** 毎秒数十回の画面更新が必須な測定において、DOMツリーのような状態を保持する描画（Retained Mode）はボトルネックです。GPUを直接駆動し、毎フレーム即座にピクセルを打つ Immediate Mode GUI (`egui`) を採用しています。
    

## 3. 技術スタック

- **Core:** Rust (Edition 2024)
    
- **Audio I/O:** `cpal` (ASIO / WASAPI Exclusive)
    
- **DSP / Math:** `rustfft`
    
- **Concurrency:** `ringbuf` (Lock-free Ring Buffer)
    
- **GUI:** `egui` + `eframe`, `egui_plot`
    

## 4. ビルドと実行要件

本システムはASIO SDKのC++ヘッダからFFIバインディングを動的に生成するため、ビルド環境に **LLVM (Clang)** が必須です。

### Windows環境でのセットアップ手順

1. **LLVMのインストール:**
    
    コマンドプロンプトまたはPowerShellから以下を実行し、LLVMをインストールします。
    
    ```
    winget install -e --id LLVM.LLVM
    ```
    
    _(※インストーラで必ず「Add LLVM to the system PATH for all users」を選択してください)_
    
2. **環境変数の設定:**
    
    `bindgen` がライブラリを捕捉できるよう、環境変数を設定します。
    
    ```
    [System.Environment]::SetEnvironmentVariable('LIBCLANG_PATH', 'C:\Program Files\LLVM\bin', [System.EnvironmentVariableTarget]::User)
    ```
    
3. **実行:**
    
    ターミナル（VSCode等）を**完全に再起動**して環境変数を反映させた後、実行してください。
    
    ```
    cargo run --release
    ```
    
    _※高フレームレート描画とDSPのゼロコスト抽象化の恩恵を得るため、必ず `--release` プロファイルで実行してください。_
    

## 5. 現在の機能 (Phase 3)

- ASIOおよびWASAPIを通じたハードウェアネイティブなオーディオストリームの構築。
    
- デバイス側のネイティブ・サンプリングレートの自動検出と、それに基づくFFT周波数分解能の動的補正。
    
- マルチチャンネル入力からの安全なLチャンネル（1ch）抽出。
    
- ロックフリーキューを経由したDSP処理と、Hanning窓適用による2048ポイントのリアルタイムFFT。
    
- GPUアクセラレーションを活用した、60fpsを保証する対数スケール振幅（dB）グラフの即時描画。
    

## 6. 今後のロードマップ (Phase 4)

測定器としての本丸である以下の機能を実装予定です。

- **デュアルチャネル同期処理:** リファレンス信号（出力）とメジャメント信号（マイク入力）の同時取得。
    
- **伝達関数（Transfer Function）:** クロススペクトル法による高解像度な振幅特性および位相（Phase）特性の算出。
    
- **コヒーレンス（Coherence）:** 測定の信頼性指標（0〜1）のリアルタイム算出。
    
- **インパルス応答（IR）:** ESS（Exponential Sine Sweep）ジェネレータとデコンボリューションの実装。
    

_このプロジェクトは、物理法則と数学的真理に基づき、妥協のない測定体験を提供します。_
